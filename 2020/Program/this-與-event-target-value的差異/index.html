This 與 event.target.value的差異

發表於 2020-11-07 更新於 2020-11-17 分類於 Program
文章字數： 596 所需閱讀時間 ≈ 1 分鐘

做範例寫 code 的時候剛好碰到 event.target.value的使用，查了一下 湊巧碰到解釋 this 與 event.target.value的差別，順手寫了一篇來記錄

根據內容用自己的語言來解釋的話， this 是具有指名意味的唯一性的

倘若 this 指定的是在input (class: target1)那麼監聽的元素就只會定位在該input上，event.target.value則不同

event.target.value的監聽過程在判斷時會有進位的狀況，照上面例子來看，假如event.target.value定位是設在div class: arron，那麼實際監聽對象會是在該元素區塊最裡層的元素，也就是 input 

只有監聽元素是非巢狀結構的單一元素時，event.target.value和this的監聽對象才會是一致的
，如果不是那麼event.target.value的監聽對象將會自動進位直到沒有下一個子元素為止。
