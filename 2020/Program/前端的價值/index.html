前端的價值

發表於 2020-11-19 分類於 Program
文章字數： 1.2k 所需閱讀時間 ≈ 2 分鐘

前言

逛著ＦＢ湊巧在 Front-End Developers Taiwan的社團裡看到這麼一篇文章，即便還沒有轉職，但對於前端還是有些想法不如就趁機寫下來紀錄一下

拆解前端與工程師

工程師

先說工程師好了，這職稱其實不難理解，利用工具（程式語言）將想法實踐出來的人，因為市場有人會為了其價值而掏錢，逐漸發展成一種工作，這類的工作古今往來都有，對於以往工匠這一詞稱就是這工作的前身了。

前端

具體去談可能有人說是 UI/UX 設計師，有說是將網頁運作實作出來的人，大部分都還是在網頁介面呈現與操作，但對於我來說前端更偏向是一個概念，手機、網頁、家電用品、燈控系統、空調系統…etc的操作介面都能規範前端的領域，一個泛指多功能系統面向使用者的最後一哩路想法。

目前大多數談到前端指的都是以網頁居多，聽到其他的說法的反而很少，在目前5G問世，6G發展的階段，隨著資料交換的速度越來越快、資料量越來越大的趨勢下，穿戴式裝置的革新依然是可預見的，ＡＲ的完整性也會越來越高，對於Siri、Alexa這類語音助理能夠做的事情也會越來越多，最終網頁功能高集成的優勢發生重心轉移的可能性是可預見的。

原文的留言裡其實有不少很有價值的留言與轉文（放在最下面），對於大部分人想像中的前端其實如原文內提到的，關於 UI/UX 設計師的重心，不管是路徑導覽、圖形色彩設計行為學上的誘導暗示，如何簡單扼要、不斷揣測使用者的資訊承受能力極限，在不影響觀感下盡可能讓植入最多訊息給使用者，已達到行銷最大化的效果。
實作方在設計師構想裏兼顧網頁渲染速度最大化、為硬體分擔風險及負荷極限的需求，盡可能將其設計原稿呈現出來，需要考量的東西之多，不亞於其他實體產品的複雜程度。

總結

實際拆解後，其實會發現與其他行業在核心概念上並沒有太多區別，所以對於一個前端而言價值是什麼？
簡單說有人願意為此掏錢那就是價值所在。

那麼前端的追求是什麼？
如同人際社會裡不可或缺的社交行為。語言交集是重中之重，書本、廣告平面、電視電台廣播等等語言傳遞都是知識、想法從腦袋掏出的一種具體展現，換言之，每個人從小在腦中就擁有專屬於自己的一本書，學習知識是將他人的書與自己的書攤開來交雜，就像電腦操作的複製貼上、取代覆蓋，融為一體後成為一本全新的書，與人交流時，各自拿著書的導覽介紹（傳達共識），找到要探討的章節之後開始執行交流的目的（研議、溝通），廣告媒體傳播就類似歇後語的碎片化訊息，補足個書本各章節缺少的微小篇幅，從而完整其行為動機，而前端做的就類似於章節導覽的工作，引導使用者找到他們想完整的章節（購物、閱讀、報稅、查詢資訊），使其動機成功得以滿足（產品推銷成功）。
如何降低從章節介紹、導覽，到使用者在縮短時間且最輕鬆的情況下翻到該章節中間所花的成本，就是前端所追求的目標。
